## 🚩CH 03 액션과 계산, 데이터의 차이를 알기

### ✔ 데이터

- 데이터란?
    - 이벤트에 대한 사실, 일어난 일의 결과

- 데이터 구현 방법?
    - JS에서는 기본 데이터 타입(숫자, 문자, 배열, 객체 등)으로 구현

- 데이터에 의미를 담는 방법?
    - 데이터 구조로 의미를 담을 수 있음
    - ex) 목록 순서가 중요한 경우 순서를 보장하는 데이터 구조를 사용(데이터 구조로 도메인 표현)

- 불변성
    - 함수형 프로그래머는 불변 데이터 구조를 만들기 위해 두 가지 원칙을 사용함
    1. 카피-온-라이트 copy-on-write. 변경할 때 복사본 만들기
    2. 방어적 복사 defensive copy. 보관하려고 하는 데이터 복사본 만들기

---

### ✔ 계산

- 계산이란?
    - 입력값으로 출력값을 만드는 것
    - 실행 시점/횟수에 관계없이 항상 같은 입력에 같은 출력 돌려줌

- 계산 구현 방법?
    - 함수

- 계산에 의미를 담는 방법?
    - 입력값을 출력값으로 만드는 것을 표현
    - 연산, 언제 어떻게 사용할지에 따라 다름

- 특징
    - 일반적으로 *'순수 함수 pure function'* 또는 *'수학 함수 mathematical function'*라고 부름
    - 계산을 쓰면서 걱정하지 않아도 되는 것
        1. 동시에 실행되는 것
        2. 과거에 실행되었던 것이나 미래에 실행할 것
        3. 실행 횟수

### ✔ 액션

- 액션이란?
    - 외부 세계에 영향을 주거나 받는 것
    - 실행 시점과 횟수에 의존함
    - 언제 실행되는지? ***순서***
    - 얼마나 실행되는지 ? ***반복***

- 액션 구현 방법?
    - JS에서 함수(계산도 함수라 구분 어려움)

- 액션에 의미를 담는 방법?
    - 액션으로 외부 세상에 영향을 줄 수 있기에 어떤 일을 하려는 지 아는 것이 중요!

- 특징
    - 일반적으로 *'순수하지 않은 함수 impure function'* 또는 *'부수 효과 함수 side-effecting function, function with side effects'*라고 부름
    - 액션은 다루기 힘들다
    - 액션은 우리가 SW를 실행하려는 가장 중요한 이유
    - 액션을 잘 사용하기 위한 방법
        1. 액션을 적게 사용(액션 대신 계산)
        2. 액션을 작게 만들기(관계 없는 코드 계산으로 빼내기)
        3. 액션이 외부 세계와 상호작용하는 것 제한
           (내부에 계산과 데이터만 있고 가장 바깥쪽에 액션이 있는 구조가 이상적임,
            어니언 아키텍처 onion architecture)
        4. 액션이 호출 시점에 의존하는 것 제한
